Задача: Построй смысловые теги по описаниям показателей из CATALOG, сгенерируй и проверь гипотезы-связки между разными тегами (liquidity, funding, capital, asset_quality, profitability, interbank, scale, efficiency, margin), оцени статус и верни ТОЛЬКО строгий JSON по схеме:
{status, confidence, reasons[], watchlist[], recommendation, metrics_snapshot, summary_ru}

Требования:
- Не опирайся на один показатель/один тег; подтверждения нужны из ≥2 независимых тегов.
- «Устойчиво» = PCT_M6 и PCT_M1 одного знака и/или ≥3 последовательные точки в одном направлении; противоположные знаки/одиночный скачок = «всплеск».
- null/отсутствующие PCT не использовать как подтверждение/опровержение; при их наличии понижай confidence.
- Учитывай компенсаторы (например, рост нормативов капитала против роста проблемных кредитов).
- В reasons каждая причина = одна связка (кросс-теги). Для каждой строки evidence укажи: КОД, направление (↑/↓), PCT_M6, PCT_M1, «устойчиво/всплеск», и дату последней точки (используй period_latest из DATA либо дату последнего значения серии).
- В metrics_snapshot включай только коды, реально использованные в reasons, с полями {latest, pct_m1, pct_m6}.
- Если данных недостаточно — "Green".
- Выводи ТОЛЬКО строгий JSON, без текста до/после.
